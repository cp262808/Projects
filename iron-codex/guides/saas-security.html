1. Multi-Tenant Isolation (12 controls)

Ensure complete isolation between tenants to prevent data leakage and unauthorized access across customer boundaries.

Implement Database-Level Tenant Isolation

Critical High Difficulty

Requirement: Implement robust database isolation using either separate schemas, databases, or row-level security policies.

Implementation Example:

-- Enable RLS on table
ALTER TABLE customer_data ENABLE ROW LEVEL SECURITY;

-- Create policy for tenant isolation
CREATE POLICY tenant_isolation ON customer_data
    FOR ALL
    TO application_role
    USING (tenant_id = current_setting('app.current_tenant')::INTEGER);

-- Set tenant context in application
SET app.current_tenant = '12345';

-- Application code (Node.js example)
const setTenantContext = (req, res, next) => {
    const tenantId = req.user.tenantId;
    req.db.query('SET app.current_tenant = $1', [tenantId], (err) => {
        if (err) return next(err);
        next();
    });
};

app.use(setTenantContext);

Validation:

  • Test cross-tenant data access attempts
  • Verify tenant context isolation
  • Validate policy enforcement

Application-Level Tenant Filtering

Critical Medium Difficulty

Requirement: Implement consistent tenant filtering in all application queries and operations.

Implementation Example:

// Base model with tenant scoping
class BaseModel extends Sequelize.Model {
    static init(attributes, options) {
        return super.init(attributes, {
            ...options,
            defaultScope: {
                where: {
                    tenantId: () => getCurrentTenant()
                }
            }
        });
    }
}

// Customer model with automatic tenant filtering
class Customer extends BaseModel {}
Customer.init({
    id: DataTypes.INTEGER,
    tenantId: DataTypes.INTEGER,
    name: DataTypes.STRING,
    email: DataTypes.STRING
}, { sequelize });

// Usage - automatically filters by current tenant
const customers = await Customer.findAll(); // Only returns current tenant's customers

Best Practices:

  • Always include tenant_id in WHERE clauses
  • Use ORM-level scoping when possible
  • Implement tenant context middleware
  • Regular security audits of queries

Network-Level Tenant Isolation

High Priority High Difficulty

Requirement: Implement network segmentation to isolate tenant traffic and prevent cross-tenant network access.

Implementation Example:

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: tenant-isolation-policy
  namespace: tenant-123
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          name: tenant-123
    - namespaceSelector:
        matchLabels:
          name: shared-services
  egress:
  - to:
    - namespaceSelector:
        matchLabels:
          name: tenant-123
    - namespaceSelector:
        matchLabels:
          name: shared-services

Security Considerations:

  • Implement defense in depth
  • Regular penetration testing
  • Monitor inter-tenant communications
  • Use encrypted network traffic

2. Data Security & Privacy (11 controls)

Protect sensitive customer data with encryption, classification, and privacy controls.

Implement Data Classification and Labeling

High Priority Medium Difficulty

Requirement: Classify and label data based on sensitivity levels and regulatory requirements.

Implementation Example:

// Data classification enum
enum DataClassification {
    PUBLIC = 'public',
    INTERNAL = 'internal',
    CONFIDENTIAL = 'confidential',
    RESTRICTED = 'restricted'
}

// Database schema with classification
CREATE TABLE customer_data (
    id SERIAL PRIMARY KEY,
    tenant_id INTEGER NOT NULL,
    data_classification VARCHAR(20) NOT NULL DEFAULT 'confidential',
    pii_fields JSONB, -- Track which fields contain PII
    created_at TIMESTAMP DEFAULT NOW()
);

// Application model with automatic classification
class CustomerData extends BaseModel {
    static associate(models) {
        // Define associations
    }
    
    // Automatic PII detection
    static async create(data, options) {
        data.pii_fields = this.detectPII(data);
        data.data_classification = this.classifyData(data);
        return super.create(data, options);
    }
    
    static detectPII(data) {
        const piiFields = [];
        if (data.email && /\S+@\S+\.\S+/.test(data.email)) piiFields.push('email');
        if (data.phone && /^\d{10}$/.test(data.phone)) piiFields.push('phone');
        if (data.ssn && /^\d{3}-\d{2}-\d{4}$/.test(data.ssn)) piiFields.push('ssn');
        return piiFields;
    }
}

Implement Field-Level Encryption

Critical High Difficulty

Requirement: Encrypt sensitive data fields using strong encryption with proper key management.

Implementation Example:

const crypto = require('crypto');
const AWS = require('aws-sdk');
const kms = new AWS.KMS();

class FieldEncryption {
    static async encrypt(plaintext, keyId, context = {}) {
        const params = {
            KeyId: keyId,
            Plaintext: Buffer.from(plaintext),
            EncryptionContext: {
                tenant: context.tenantId,
                field: context.fieldName
            }
        };
        
        const result = await kms.encrypt(params).promise();
        return result.CiphertextBlob.toString('base64');
    }
    
    static async decrypt(ciphertext, context = {}) {
        const params = {
            CiphertextBlob: Buffer.from(ciphertext, 'base64'),
            EncryptionContext: {
                tenant: context.tenantId,
                field: context.fieldName
            }
        };
        
        const result = await kms.decrypt(params).promise();
        return result.Plaintext.toString();
    }
}

3. Access Control & IAM (10 controls)

Implement comprehensive access controls with role-based permissions and multi-factor authentication.

Implement Role-Based Access Control (RBAC)

High Priority Medium Difficulty

Requirement: Define granular roles and permissions for different user types across tenants.

Implementation Example:

// Permission-based authorization
const permissions = {
    'users:read': 'Read user information',
    'users:write': 'Create and update users',
    'users:delete': 'Delete users',
    'reports:read': 'View reports',
    'reports:write': 'Create reports',
    'admin:manage': 'Administrative functions'
};

// Role definitions
const roles = {
    'viewer': ['users:read', 'reports:read'],
    'editor': ['users:read', 'users:write', 'reports:read', 'reports:write'],
    'admin': ['users:read', 'users:write', 'users:delete', 'reports:read', 'reports:write', 'admin:manage'],
    'tenant_admin': ['users:read', 'users:write', 'users:delete', 'reports:read', 'reports:write']
};

// Middleware to check permissions
const requirePermission = (permission) => {
    return (req, res, next) => {
        const userRoles = req.user.roles || [];
        const userPermissions = userRoles.flatMap(role => roles[role] || []);
        
        if (!userPermissions.includes(permission)) {
            return res.status(403).json({ error: 'Insufficient permissions' });
        }
        
        next();
    };
};

// Usage
app.get('/api/users', requirePermission('users:read'), getUsersHandler);
app.post('/api/users', requirePermission('users:write'), createUserHandler);
