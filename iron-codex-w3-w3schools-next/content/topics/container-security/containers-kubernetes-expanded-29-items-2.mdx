<summary>Kubernetes → Admission Control &amp; Policy</summary>
<p class="tags"><b>Tags:</b> Admission, OPA, Kyverno</p>
<p><b>Question:</b> Do admission controllers (OPA/Kyverno) enforce pre-deploy security policies?</p>
<p><b>Applicable Requirements:</b><br/>- NIST 800-53: CM-7, SI-10<br/>- CIS Benchmark: Kubernetes</p>
<p><b>Applicability:</b> K8s clusters any cloud/on‑prem</p>
<p><b>Expected Result:</b> Block privileged pods, hostPath, :latest images, missing probes; require signed images.</p>
<p><b>Why It Matters:</b> Stops risky workloads at the gate.</p>
<p><b>Technical Breakdown:</b></p>
<ol>
<li>Deny default SA mounts; require limits/requests.</li>
<li>Break-glass labels with approvals.</li>
</ol>
<details><summary>Deep Dive</summary><ul><li>Kubernetes (Kyverno): require signed images; (Istio): `PeerAuthentication` with `mtls: STRICT`; (kubectl): `kubectl get networkpolicy -A` to verify default deny.</li><li><a href="https://kubernetes.io/docs/concepts/security/pod-security-admission/" target="_blank">Kubernetes: Pod Security Admission</a></li><li><a href="https://kubernetes.io/docs/tasks/debug/debug-cluster/audit/" target="_blank">Kubernetes: Audit Logging</a></li><li><a href="https://docs.sigstore.dev/cosign/overview/" target="_blank">Sigstore Cosign: Container signing</a></li><li><a href="https://kyverno.io/policies/" target="_blank">Kyverno Policy Examples</a></li><li><a href="https://github.com/open-policy-agent/gatekeeper-library" target="_blank">OPA Gatekeeper Library</a></li></ul></details></details>
