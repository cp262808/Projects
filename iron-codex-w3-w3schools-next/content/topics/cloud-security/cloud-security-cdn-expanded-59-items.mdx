<summary>Cloud Security (AWS) → Storage Public Access</summary>
<p class="tags"><b>Tags:</b> Storage, Public Access</p>
<p><b>Question:</b> Are object/blob buckets private with org-level blocks and justified exceptions?</p>
<p><b>Applicable Requirements:</b><br/>- NIST 800-53: AC-3, SC-7<br/>- CIS Benchmark: Cloud Storage</p>
<p><b>Applicability:</b> Object/blob storage</p>
<p><b>Expected Result:</b> Public access blocks; no public ACLs; access via signed URLs or IAM only.</p>
<p><b>Why It Matters:</b> Misconfig drives many breaches.</p>
<p><b>Technical Breakdown:</b></p>
<ol>
<li>Enable account-level blocks; Config/Policy scans; alert on drift.</li>
</ol>
<details><summary>Deep Dive</summary><ul><li>AWS CLI: `aws s3api put-public-access-block --bucket &lt;name&gt; --public-access-block-configuration BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true`</li></ul></details></details>
